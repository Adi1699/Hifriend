# OrangeHRM Test Cases

## Preconditions
- Application URL: [Login Page](https://opensource-demo.orangehrmlive.com/web/index.php/auth/login)
- Valid credentials: 
  - **Username**: Admin
  - **Password**: admin123

## Test Case 1: Access Global Header and Navigation
- **Description**: Verify elements in the global header and navigation.
- **Steps**:
  1. Login to the application with valid credentials.
  2. Verify presence of OrangeHRM logo linking to the Dashboard.
  3. Check for the Upgrade button and its functionality.
  4. Verify user profile section options like My Profile, Change Password.
  5. Validate presence of Help and More Dropdown icons.
- **Expected Result**: All elements should display as per the UI design.

## Test Case 2: Dashboard Widgets Functionality
- **Description**: Validate the functionality of each widget on the Dashboard.
- **Steps**:
  1. Navigate to Dashboard upon login.
  2. Verify each widget like Time at Work, My Actions, Quick Launch, Buzz Latest Posts, etc.
  3. Check interactive elements like buttons and links redirect correctly.
- **Expected Result**: Widgets display accurate data, and links are functional.

## Test Case 3: New Job Section under Admin Module
- **Description**: Validate the new Job Section in Admin Module.
- **Steps**:
  1. Navigate to Admin → Job.
  2. Check list of existing job titles.
  3. Use Add New to create a job title and fill out fields.
  4. Save the new record and ensure a confirmation message appears.
- **Expected Result**: Users can manage job titles efficiently with validation checks.

## Test Case 4: User Type Field in Edit User Section
- **Description**: Validate the new User Type field in Edit User process.
- **Steps**:
  1. Navigate to Admin/User Management.
  2. Select a user and go to Edit User.
  3. Check User Type field is present and mandatory.
  4. Save changes with valid selection ensuring no errors.
- **Expected Result**: The field should enforce role selection and save successfully.
